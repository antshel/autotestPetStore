## API автоматизация (PetStore)

Автоматизированные тесты для REST API сервиса Swagger Petstore.

### Используемые технологии

* Python
* pytest
* requests
* REST API

### Тестируемый API
[Swagger Petstore](https://petstore.swagger.io/#/)

### Покрытые тестовые сценарии

### Покрытые тестовые сценарии

#### Создание животного — `POST /pet`

| Тип сценария | Описание |
|---|---|
| Позитивный | создание животного с валидными данными |
| Негативный | создание животного с некорректным типом `id` |
| Негативный | отправка пустого тела запроса |
| Негативный | создание животного без обязательного поля `name` |
| Проверка | статус-код ответа |
| Проверка | корректность имени животного |
| Проверка | корректность статуса животного |

---

#### Получение животного — `GET /pet/{petId}`

| Тип сценария | Описание |
|---|---|
| Позитивный | получение животного по существующему `id` |
| Негативный | получение животного по несуществующему `id` |
| Негативный | получение животного по строковому `id` |
| Проверка | статус-код ответа |
| Проверка | соответствие `id` в ответе |

---

#### Обновление животного — `PUT /pet`

| Тип сценария | Описание |
|---|---|
| Позитивный | обновление имени и статуса животного |
| Проверка | статус-код ответа |
| Проверка | корректность `id` |
| Проверка | обновление имени |
| Проверка | обновление статуса |

---

#### Удаление животного — `DELETE /pet/{petId}`

| Тип сценария | Описание |
|---|---|
| Позитивный | удаление существующего животного |
| Негативный | удаление несуществующего животного |
| Проверка | статус-код ответа |
| Проверка | проверка отсутствия объекта после удаления |

### Используемые подходы

* параметризация тестов (pytest.mark.parametrize)
* использование pytest fixtures
* разделение API методов и тестов (endpoint pattern)
* вынесение тестовых данных в отдельный модуль

### Установка зависимостей

```pip install -r requirements.txt```

### Запуск тестов

```pytest -v```
