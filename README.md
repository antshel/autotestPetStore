## API автоматизация (PetStore)

Автоматизированные тесты для REST API сервиса Swagger Petstore.

### Используемые технологии

* Python
* pytest
* requests
* REST API

### Тестируемый API
[Swagger Petstore](https://petstore.swagger.io/#/)

### Покрытые тестовые сценарии

### Создание животного — `POST /pet`

| Тип | Сценарий | Проверки | Название теста |
|------|----------|----------|----------------|
| ✅ Позитивный | Создание животного с валидными данными | Статус-код 200, поля id, name, status в ответе | `test_create_pet_success` |
| ❌ Негативный | Создание с некорректным типом `id` (string) | Статус-код 500 | `test_create_invalid_id_type` |
| ❌ Негативный | Отправка пустого тела запроса | Статус-код 405 | `test_empty_body` |
| ❌ Негативный | Создание без обязательного поля `name` | Статус-код 405 | `test_create_without_name` |

### Получение животного — `GET /pet/{petId}`

| Тип | Сценарий | Проверки | Название теста |
|------|----------|----------|----------------|
| ✅ Позитивный | Получение животного по существующему `id` | Статус-код 200, id совпадает с запрошенным | `test_get_pet_by_id` |
| ❌ Негативный | Получение по несуществующему `id` | Статус-код 404 | `test_get_pet_by_invalid_id` |
| ❌ Негативный | Получение по строковому `id` | Статус-код 404 | `test_get_pet_by_string_id` |

### Обновление животного — `PUT /pet`

| Тип | Сценарий | Проверки | Название теста |
|------|----------|----------|----------------|
| ✅ Позитивный | Обновление данных животного (имя и статус) | Статус-код 200, id, name, status в ответе | `test_update_pet` |

### Удаление животного — `DELETE /pet/{petId}`

| Тип | Сценарий | Проверки | Название теста |
|------|----------|----------|----------------|
| ✅ Позитивный | Удаление существующего животного | Статус-код 200 | `test_delete_pet` |
| ✅ Позитивный | Проверка отсутствия после удаления | GET запрос возвращает 404 | `test_delete_pet` |
| ❌ Негативный | Удаление несуществующего животного | Статус-код 404 | `test_delete_non_existent_pet` |

### Используемые подходы

* параметризация тестов (pytest.mark.parametrize)
* использование pytest fixtures
* разделение API методов и тестов (endpoint pattern)
* вынесение тестовых данных в отдельный модуль

### Установка зависимостей

```pip install -r requirements.txt```

### Запуск тестов

```pytest -v```
